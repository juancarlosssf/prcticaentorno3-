# Práctica 1 - Checklist de Entrega

## 1) Fase 1 - Preparación del entorno

- [ ] IDE con IA instalado (Cursor/Copilot equivalente).
- [ ] Captura 1: autocompletado en línea activo.
- [ ] Captura 2: chat contextual sobre el proyecto.
- [ ] Documento creado: `REGISTRO_CONFIGURACION.md`.

## 2) Fase 2 - Script asistido por IA

- [x] Script creado por prompteado: `validador.js`.
- [ ] Archivo de prueba: `socios.csv`.
- [ ] Capturas de diff (verde) aceptando/rechazando cambios (mínimo 3).

## 3) Fase 3 - Auditoría de alucinación

- [ ] Forzar o detectar un fallo lógico de la IA.
- [ ] Documentar por qué falló (párrafo técnico).
- [ ] Mostrar prompt exacto de corrección.
- [ ] Confirmar solución final funcional.
- [ ] Documento creado: `AUDITORIA_FALLO_IA.md`.

## 4) Evidencias obligatorias de entrega

- [ ] Historial de prompts (capturas o transcripción amplia).
- [ ] 3 capturas de diff del editor (additions en verde).
- [ ] Informe de auditoría del fallo.
- [ ] URL de repositorio Git público.

## 5) Empaquetado final

- [ ] Exportar a PDF/ZIP todo el material.
- [ ] Revisar que las capturas sean legibles.
- [ ] Verificar que la URL pública funcione.
